# OrgMap — الخارطة التنظيمية التفاعلية

نسخة GitHub Pages ثابتة لعرض الخارطة التنظيمية، لوحة القيادة، تقارير السعودة، فحص جودة البيانات، وتحويل قالب Excel إلى ملف البيانات `data.js`.

---

## الملفات الرئيسية

| الملف | الوصف |
|---|---|
| `index.html` | لوحة القيادة الرئيسية والمؤشرات العامة والتحليلات البصرية |
| `org.html` | صفحة الهيكل التنظيمي التفاعلي |
| `saudization.html` | تقرير السعودة حسب الشركة والإدارات والأقسام |
| `quality.html` | فحص جودة البيانات والتحذيرات |
| `compare.html` | مقارنة نسخة قديمة ونسخة جديدة من `data.js` |
| `convert-ultra-easy-excel-to-data.html` | تحويل قالب Excel إلى ملف `data.js` |
| `admin.html` | أداة Admin محلية مخفية لتعديل البيانات وتوليد `data.js` |
| `data.js` | ملف البيانات الرئيسي للنظام |
| `org-template-ultra-easy-hr.xlsx` | قالب Excel الرسمي لإدخال البيانات |
| `images/` | مجلد صور الموظفين والإدارات والشعارات |
| `logo.png` | شعار الشركة المستخدم في الواجهة |
| `ologo.png` | شعار الشركة داخل الهيكل |
| `favicon.png` | أيقونة تبويب المتصفح |

## تعديل البيانات

مصدر البيانات الرئيسي هو:

```text
data.js
```

لا يتم تعديل البيانات داخل `index.html`.

يمكن تحديث `data.js` بثلاث طرق:

### 1. عبر Admin المحلي

افتح يدويًا:

```text
admin.html
```

هذه الصفحة غير موجودة في القائمة الرئيسية عمدًا، لأنها للاستخدام الداخلي فقط.

من خلالها يمكن:

- إضافة موظف
- حذف موظف
- تعديل موظف
- تعديل الراتب
- تعديل الجنسية
- تعديل التبعية
- تعديل الصورة
- تعديل الحالة
- تعديل بيانات الإصدار:
  - الإصدار
  - آخر تحديث
  - ملاحظات الإصدار
- تحميل ملف `data.js` جديد

بعد تحميل `data.js` الجديد، استبدل الملف القديم به ثم اختبر محليًا.

### 2. عبر قالب Excel

استخدم الملف:

```text
org-template-ultra-easy-hr.xlsx
```

ثم افتح:

```text
convert-ultra-easy-excel-to-data.html
```

وارفع القالب بعد تعبئته. الأداة تولد لك ملف `data.js`.

### 3. تعديل يدوي مباشر

يمكن تعديل `data.js` يدويًا عند الحاجة، لكن الأفضل استخدام Admin أو Excel لتقليل الأخطاء.

## قواعد مهمة في البيانات

### الشواغر

أي عنصر فيه:

```js
status: "vacant"
```

أو:

```js
tone: "vacant"
```

يظهر في الهيكل كمنصب شاغر، لكنه لا يدخل في:

- عدد الموظفين
- عدد المدراء
- عدد المدراء التنفيذيين
- تكاليف الرواتب
- تحليلات السعودة
- مراكز التكلفة

الموظفون الحقيقيون تحت المنصب الشاغر يستمر احتسابهم طبيعيًا.

---

### مراكز التكلفة

تقرير أعلى مراكز التكلفة يعتمد على:

```js
is_cost_center: true
```

العناصر التي لا يجب أن تدخل كمراكز تكلفة تكون:

```js
is_cost_center: false
```

مثال:

```js
{
  name: "ادارة تقنية المعلومات",
  is_cost_center: true
}
```

و:

```js
{
  name: "الادارة التنفيذية",
  is_cost_center: false
}
```

---

### الجنسية والسعودة

يتم استخدام الحقل:

```js
nationality_group: "saudi"
```

أو:

```js
nationality_group: "non_saudi"
```

القيم تظهر في:

- لوحة القيادة
- تقرير السعودة
- كروت الموظفين
- فحص جودة البيانات

السعودي يظهر كبادج:

```text
🇸🇦 سعودي
```

وغير السعودي يظهر كبادج:

```text
🌐 غير سعودي
```

## إخفاء التكاليف

بعض التكاليف تكون مخفية افتراضيًا، وتظهر بعد إدخال الرمز داخل الواجهة.

الهدف من ذلك حماية البيانات المالية عند عرض النظام على شاشة عامة.

---

## الصور

ضع صور الموظفين أو الإدارات داخل:

```text
images/
```

ثم اكتب اسم الصورة داخل البيانات، مثل:

```js
image: "ali.png"
```

إذا لم توجد صورة، يظهر رمز افتراضي حسب نوع العنصر.

## الصفحات

### لوحة القيادة

```text
index.html
```

تعرض:

- عدد الموظفين
- عدد الإدارات والأقسام
- عدد المدراء
- عدد المدراء التنفيذيين
- عدد المحطات المملوكة
- عدد المحطات المشغلة
- عدد العاملين بالمحطات
- عدد الموظفين الإداريين
- أعلى الإدارات والأقسام تكلفة
- تحليلات السعودة
- التحليل البصري للمؤشرات التشغيلية

### الهيكل التنظيمي

```text
org.html
```

يعرض الهيكل التنظيمي التفاعلي مع:

- فتح الكل
- طي الكل
- وضع التركيز
- إظهار / إخفاء الرواتب
- عرض تنفيذي
- لوحة تفاصيل جانبية
- Badges للجنسية

### تقرير السعودة

```text
saudization.html
```

يعرض:

- توزيع السعوديين وغير السعوديين
- نسبة السعودة
- السعودة حسب الإدارات والأقسام
- تكلفة السعوديين وغير السعوديين
- Pie Chart و Bar Chart

### فحص جودة البيانات

```text
quality.html
```

يعرض تحذيرات مثل:

- موظف بدون جنسية
- موظف بدون راتب
- إدارة فيها بيانات ناقصة
- بيانات سعودة غير مكتملة

### مقارنة النسخ

```text
compare.html
```

تستخدم لمقارنة ملف `data.js` قديم مع ملف جديد.

### تحويل Excel

```text
convert-ultra-easy-excel-to-data.html
```

تحول قالب Excel إلى `data.js`.

### Admin المحلي

```text
admin.html
```

أداة تعديل محلية مخفية وغير مربوطة بالقائمة.
